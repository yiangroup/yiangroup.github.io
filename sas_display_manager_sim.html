Editor

Editor - write SAS code here

The Editor is where a programmer writes or pastes SAS code. In Display Manager, you can submit selected code or submit the whole editor.

  • Shortcut: F8 submits code.
  • Risk: if nothing is selected, SAS may run more code than intended.
  • Habit: run a small section, then read the Log.
Log

Log - read this after every run

The Log is the execution record. It tells you whether SAS created data, how many observations were processed, and whether there are problems.

  • NOTE normal execution details.
  • WARNING code ran, but review is required.
  • ERROR code failed or output is not reliable.
Explorer

Explorer - browse libraries and data

Explorer is the data/file browser. Beginners should learn where WORK, permanent libraries, and shortcuts appear.

  • WORK: temporary library cleared after the session.
  • Permanent libraries: assigned with LIBNAME.
  • Viewtable: useful for quick inspection, not formal QC.
Toolbar

Toolbar - common actions

The toolbar provides quick access to open, save, print, submit, cancel, and help actions. The green run icon is the key beginner action.

  • Submit: run selected code or the active editor.
  • Cancel: stop submitted statements.
  • Save: save code as a .sas file.
Command Box

Command box - jump to windows

The command box accepts short commands in Display Manager. Try typing log, editor, output, or explorer.

  • log: show the Log window.
  • editor: show the Editor window.
  • output: show listing output.
Output

Output - review printed results

The Output window shows classic listing output produced by procedures such as PROC PRINT, PROC FREQ, and PROC MEANS. Some SAS sessions also show results in a Results Viewer.

  • Use it for: quick review of procedure output.
  • Do not rely on it alone: still check the Log for errors and warnings.
  • Production habit: capture key procedure output with ODS when it needs QC.
