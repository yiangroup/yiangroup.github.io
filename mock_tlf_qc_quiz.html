Scenario 1: Dual-Visit Windowing Trap

Subject 101-002-015 has two unscheduled SBP records inside the Week 12 window: Study Day 80 and Study Day 94.

Blueprint: One analysis visit should have one selected analysis record per subject/parameter unless the specification says otherwise. Check the ADaM BDS visit window and tie-break rule, then derive one Week 12 ADVS record.

Scenario 2: Crude vs Model-Based Row Trap

The shell contains both crude descriptive rows and ANCOVA statistical analysis rows. A draft output programmer plans to populate the entire table from the ANCOVA output dataset because it already contains LS means and p-values.

Blueprint: TLF shells often mix descriptive statistics and inferential model results. Crude rows should be summarized directly from the analysis dataset, while ANCOVA rows come from the model output specified in the SAP.

Scenario 3: P-value Placement Trap

The shell shows a p-value row under both treatment columns, but the footnote says the p-value comes from an ANCOVA treatment comparison.

Blueprint: A treatment-comparison p-value is not arm-specific. Confirm the shell layout with the statistician; display a single p-value column or a clearly labeled comparison row.
